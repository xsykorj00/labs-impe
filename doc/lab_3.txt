3.  Pulse-width modulation (PWM) using a timer and an application built on top of it
    (controlling the brightness of an RGB LED).

    • Assignment details; when solving this task, take a look on the Chapter 29 Timer/PWM
      Module (TPM) and the Laboratory Kit Schematic (e.g. to identify the
      timer output pins (channels) that can be used to control the LED via a PWM signal
      from the MCU interface):

        o After programming and running the embedded application will enter the infinite
          loop in main(), where it will first increment the variable compare up to the final
          value OVERFLOW, and then decrement it back towards 0 (note: switching between
          increment/decrement modes is controlled by the variable increment). This behaviour
          will result in the illusion of breathing the light.

        o Complete the template embedded application, i.e. the bodies of the functions
          Timer0Init() and main(), so that the resulting application behaves according
          to the following as expected (see also comments in C template):

            - The application will cyclically perform the following: first gradually
              increase, and after reaching the maximum, gradually decrease the brightness
              of the three-colour (RGB) LED. To set a specific brightness level on each
              of the R, G, B channels, use the appropriate PWM signal generated by the
              timer available on the MCU. Your task is to implement mussing parts of these
              functions:
              + void TPM0_IRQHandler(void)
              + static void tpm0_init(void)
              + int main(void)
